# Bellog

**Beautify and filter your Logs and Serial streams** ✨

## Introduction

Bellog is a tool designed to beautify and filter any stream of text by setting a set of rules.

You can turn the usual ugly log file generated by a software, operative system or peripheral into an easier-to-read format and also apply logic to analyse logs of the same kind.

Bellog supports:

- Reading data from **clipboard**
- Reading data from **file** (no live monitoring)
- Connecting to a **TCP Server** (via Websockify — see the [guide](documentation/Websockify.md))
- Connecting to **COM ports** via [WebSerial API](https://developer.mozilla.org/en-US/docs/Web/API/Web_Serial_API) (Chrome / Edge)
- Connecting to **CAN Bus** adapters via WebSerial (slcan) or WebSocket (SocketCAN bridge)
- Connecting to **ADB** devices and running logcat (Chrome / Edge)
- Storing logs locally in the browser (IndexedDB)

> **Browser support:** Google Chrome or Microsoft Edge are required for WebSerial and ADB.
> Other Chromium derivatives may not work (e.g. Brave does not expose the WebSerial API).

## Samples

Built-in sample profiles are available inside the application under **Import by Example**:

| Sample | Description |
|--------|-------------|
| Clipboard Line Viewer | Paste text, see each line with timestamps |
| JSON Lint | Paste JSON to validate and pretty-print |
| Serial Port Logger | WebSerial line-based log viewer with send capability |
| IoT Device Logger | Serial log with CBOR→JSON decoding, error highlighting, allocation tracking |
| CANopen Decoder | CAN Bus → CANopen protocol decoder with colour-coded NMT, SDO, PDO, HB, EMCY views |

## Security

Bellog processes everything locally — no data leaves your browser.
However, profiles (`.bll` files) and libraries execute JavaScript at runtime.
**Only import profiles from sources you trust.**

Read the full [Security Considerations](documentation/SecurityConsiderations.md).
